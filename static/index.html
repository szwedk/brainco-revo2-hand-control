Revo2Touch Dev Console

A real-time control and data platform for the BrainCo Revo2Touch prosthetic hand. It streams motor positions, tactile force data, and proximity readings over WebSocket at ~25 Hz and lets you control, record, classify, and replay hand gestures — all in-browser.

⚡The server runs locally at http://localhost:8765. Start it with python server.py from the brainco/ folder. The UI auto-reconnects if the connection drops.
Feature Overview
🤚 Tactile Sensors
Per-finger normal force, tangential force, direction vector, and proximity with live arc gauges.
🎮 Finger Control
6 independent sliders (Thumb, ThAux, Index, Middle, Ring, Pinky) 0–1000. Drag, scroll, or click the SVG hand.
🖐️ Pose Library
12 built-in poses. One-click or keyboard shortcut. Run Demo cycles all poses automatically.
📹 Mirror Mode
MediaPipe hand tracking mirrors your webcam hand posture onto the robot in real time.
🧠 k-NN Classifier
Train an in-browser k-nearest-neighbors model on labeled snapshots. Live prediction at full stream rate.
⏱️ Motion Replay
Record continuous hand motion frame-by-frame. Replay at adjustable speed.
🔔 Force Alerts
Per-finger threshold alerts with visual indicator and optional audio beep.
📊 PCA Scatter
2D PCA projection of labeled gesture samples for visual class separability analysis.
⬇️ Export
Buffer, sessions, and labels exportable as JSON or CSV. WebSocket URL for external Python/Node clients.
UI Layout

The interface is split into three resizable columns and a collapsible bottom drawer. All dividers can be dragged.

Columns
L
Left column — Tactile Sensors (top) and Finger Control sliders (bottom). Default width 380 px, draggable up to 700 px.
C
Center column — 3D SVG hand visualization. Fingers are interactive: click to toggle, scroll to set value, drag up/down to scrub. Heatmap blobs show live force per fingertip.
R
Right column — top half holds Poses panel + Custom Code editor side by side; bottom half is the Force History chart (switchable to Radar view).
Bottom Drawer

Drag the thin handle above the drawer to resize it. Click Collapse to hide. Tabs: Stream Inspector, Data Logger, Session Recorder, Labels, Export, Classifier, Sequencer, Motion Rec, Alerts, PCA, Key Map, Help.

Header Bar

Shows connection status, device info, live Hz, frame count, and session timer. Record starts a session recording directly. Mirror activates webcam hand tracking.

Tactile Sensors

Five force sensors (Thumb through Pinky) stream data at ~25 Hz. Each card shows:

1
Arc gauge — fill proportion = normal force / 2500 N scale. Color shifts green → yellow → red as force increases.
2
Large number — raw normal force in Newtons.
3
Compass arrow — tangential force direction. Rotates to show shear direction.
4
Proximity bar — capacitive proximity (0–200 range). Useful for pre-contact detection.
5
Status dot — green = OK, yellow = warning, red = error from SDK status byte.
💡The heatmap overlay on the SVG hand visualizes the same normal force data — fingertips glow green→red in real time. No interaction required.
Finger Control

Six motor channels: Thumb, ThAux, Index, Middle, Ring, Pinky. Values 0 (open) to 1000 (fully closed).

Ways to set a value
A
Tile slider — drag the thin bar at the bottom of each finger tile.
B
SVG hand drag — click and drag a finger segment up or down. Delta × 4 per pixel.
C
SVG hand scroll — hover a finger and scroll ±50 per notch.
D
SVG hand click — toggles between 0 and 1000 (tap to open/close).
E
Custom Code API — call send([t,ta,i,m,r,p]) from the code editor.
⚠️A 1.5-second slider lock prevents incoming WebSocket data from overriding manual controls. After 1.5 s of inactivity the live stream resumes updating sliders.

Sequencer (drawer tab)
1
Select a pose and duration (ms) in the Add step row, then click + Add. Or use + All poses to add every pose at once.
2
Click ▶ Play. Each step highlights as active and shows a progress bar. Enable Loop to repeat forever.
3
The Tween speed slider controls how many interpolation steps are applied per animation frame when transitioning between poses.
🎵Run Demo (in the Poses panel) is a quick preset sequence. For custom sequences use the Sequencer tab.
Custom Code

Write JavaScript that runs directly in the browser and controls the hand via a small async API.

API Reference
→
send([t, ta, i, m, r, p]) — send 6 motor positions (0–1000) immediately.
→
pose('fist') — send a named pose from the built-in library.
→
await sleep(ms) — async delay. Must be used inside an async function.
→
log('message') — print to the output area below the editor.
🔧The code runs in 'use strict' mode. Any uncaught error is shown in the output area prefixed with ERR:. Click Clear to reset the output.
Mirror Mode

Uses Google MediaPipe Hands to detect your hand from the webcam and mirror the curl of each finger onto the robot in real time.

1
Click Mirror in the header. Grant camera permission when prompted.
2
A PiP (picture-in-picture) video appears bottom-right of the hand visualization with a skeleton overlay.
3
Your finger curls are mapped 0→1000. An exponential moving average (α=0.35) smooths the output.
4
Click Stop Mirror to release the camera. Slider lock is activated while mirror is running.
💡Best results in good lighting with your palm facing the camera. MediaPipe requires a HTTPS or localhost origin — this app runs on localhost so it works out of the box.
Recording & Labels
Session Recorder

Captures full-rate frames (positions + force) into a named session that can be downloaded as JSON. Up to 18,000 frames are kept in the live buffer at all times.

1
Go to Session Recorder tab (or click the Record button in the header) and click ● Start Recording.
2
Perform your gestures. Frame count and duration update live.
3
Click ■ Stop. The session appears in the list. Download or delete it anytime.
Labels (Snapshots)

Capture a single-frame snapshot with a text label. Used as training data for the k-NN classifier.

1
Go to the Labels tab. Type a label name (e.g. open, grasp_cup).
2
Press Enter or click + Capture. The current motor positions and force levels are saved.
3
Capture 5–10 samples per class for best classifier accuracy. Export as CSV for offline ML training.
Gesture Classifier (k-NN)

An in-browser k-nearest-neighbors classifier trained on labeled snapshots. It runs at stream rate (~25 Hz) and shows the predicted gesture with confidence.

1
Capture labeled samples (at least 2 different labels, 3+ samples each) in the Labels tab.
2
Open the Classifier tab. Set k (default 3) and click ↻ Train k-NN.
3
The large prediction label updates live. The confidence bar and candidate list show vote distribution across classes.
4
Open the PCA tab to see a 2D scatter of your samples — overlapping clusters mean the classes are hard to separate.
🧠Feature vector is the 6 motor positions normalised 0–1. More diverse sample positions per class → better accuracy. k=1 is fastest but overfit; k=5–7 gives smoother predictions.
Motion Recorder & Replay

Records a time-series of motor positions (not just a snapshot) and replays them with accurate timing.

1
Open the Motion Rec tab. Click ● Record Motion.
2
Control the hand however you like — sliders, poses, custom code, mirror mode. Every incoming frame is recorded.
3
Click ■ Stop. The motion appears in the list.
4
Click ▶ to replay. Adjust Playback speed (0.1× slow-mo to 3× fast). A progress bar tracks replay position.
⏱️Playback uses wall-clock interpolation — actual frame timing is preserved regardless of browser framerate.
Force Threshold Alerts

Monitor per-finger force and get notified when a threshold is exceeded.

1
Open the Alerts tab. Each finger has a threshold slider (50–2500 N).
2
Enable/disable individual fingers with the ON/OFF toggle. The red indicator dot pulses when firing.
3
Enable the Sound checkbox for an audio beep (different pitch per finger). Click Test Beep to preview.
🔧Useful for grasp calibration — set the threshold to the maximum safe grip force and monitor during object manipulation.
Export & External API
Export formats
→
Buffer → JSON/CSV — last 18,000 frames with timestamps, positions, and force values.
→
Labels → CSV — snapshot dataset ready for pandas / scikit-learn.
→
Sessions → JSON — all recorded sessions merged.
WebSocket API

Connect directly to ws://localhost:8765/ws from any client.

Incoming frames: { connected, positions[6], touch[5], ts, device_info }

Send pose: { type: "pose", name: "fist" }

Send positions: { type: "set_positions", positions: [0,0,500,500,500,500] }

🐍Use the Python Snippet button in Export to copy a ready-to-run asyncio consumer.
